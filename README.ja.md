# Diff History

> [English version](README.md)

ファイルの差分を自動保存し、Git風のdiffビューアで過去のバージョンを確認・復元できるObsidianプラグインです。モバイルでも動作します。

## 機能

- **自動diff追跡** — Markdownファイルの編集時に差分（パッチ）を自動保存。変更部分のみを保存するため、ストレージ効率が良い。
- **履歴サイドバー** — 日付ごとにグループ化された変更履歴を表示。タイムスタンプと行の変更統計（+追加 / -削除）付き。
- **サイドバイサイドdiffビューア** — 連続するスナップショット間の差分を色分け表示（追加=緑、削除=赤）。スクロール同期、行番号付き。
- **ワンクリック復元** — 任意の時点にファイルを復元。復元前の状態は新しい履歴エントリとして保存される。
- **コンテキストメニュー** — Markdownファイルを右クリックしてdiff履歴を表示。

## 設定

| 設定 | デフォルト | 説明 |
|------|-----------|------|
| Retention period | 7日 | 履歴の保持期間。超えた記録は自動削除 |
| Debounce interval | 5秒 | 最後の編集からdiff保存までの待機時間 |
| Minimum interval | 60秒 | 同一ファイルの連続保存間の最小間隔 |
| Exclude patterns | (なし) | 除外するファイル/フォルダのGlobパターン（例: `templates/**`） |
| Max storage | 200 MB | IndexedDBの最大使用量 |

## コマンド

- **Show file history** — アクティブファイルの履歴サイドバーを開く
- **Restore to point** — アクティブファイルを過去の状態に復元
- **Clear current file history** — アクティブファイルの履歴を全削除

## 仕組み

1. Markdownファイルの編集を検知し、デバウンスで待機
2. 待機後、前回の状態との差分（パッチ）を計算してIndexedDBに保存
3. ファイルの初回キャプチャはinitial commitとして記録（空文字列との差分）
4. 履歴サイドバーには行レベルの正確な変更数を表示（gitと同じカウント方式）
5. 「Diff」ボタンで連続するスナップショット間の差分を表示
6. 「Restore」ボタンでその時点のファイルを復元
7. 保持期間・ストレージ上限に基づいて古い記録を自動クリーンアップ

## 技術詳細

- ストレージ: IndexedDB（[Dexie](https://dexie.org/) 経由）
- 差分アルゴリズム: [diff-match-patch](https://github.com/google/diff-match-patch)
- 行数カウントは行レベルdiffを使用（gitと一致）
- ファイル復元は最新スナップショットからパッチを逆適用して再構築
